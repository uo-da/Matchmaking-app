# CLAUDE.md

## プロジェクト概要
- [プロジェクト名]: エンジニア向けマッチングアプリ
- 技術スタック: JavaScript / React / WebSocket / ブラウザごとのローカルストレージ

## 機能要件
- GitHub連携ログイン
- 年齢確認 (18歳以上のみ)
- プロフィール登録・編集
- プロフィール画面でGitHubの草が見れる
- ユーザ一覧・検索: 技術タグや経験年数でのフィルタリング
- 「いいね」とマッチング: 互いに「いいね」をしたらマッチ成立
- シンプルチャット: マッチング成立者とのリアルタイムメッセージ

## 非機能要件
- スーパーライク機能
- プッシュ通知（または未読バッジ）：交流を促進する仕組み
- おすすめユーザ表示: 自分の技術スタックと相性の良い人を自動提示
- マッチング相手と共同作業ができるエディタ

## 仕様
- TinderのようなUI
- プロフィールに年齢・自己紹介・技術スタック・趣味
- チャットとテキストエディタはWebSocketで

## コーディング規約
- 言語: JavaScript（フロントエンド）
- すべての関数にJSDocコメントを記載すること
- テストはJestで記述
- 変数名: camelCase
- インデント: スペース2つ

## ファイル構成
- src/components/ - Reactコンポーネント
- src/hooks/ - カスタムフック
- src/utils/ - ユーティリティ関数
- src/services/ - APIサービス
- tests/ - テストコード

## セキュリティルール（最重要）
- GitHub認証は必ずOIDCを使用
- Secretsは GitHub Secrets に格納
- ユーザデータは暗号化して保存

## デプロイフロー
1. featureブランチで開発
2. PRを作成 → CI（テスト + lint）が自動実行
3. mainマージ → staging自動デプロイ
4. タグ付け → production手動デプロイ

## 命名規則
- コンポーネント: PascalCase
- 関数: camelCase
- 定数: UPPER_SNAKE_CASE

## 優先順位（上が最優先）
1. セキュリティ（認証・認可・シークレット管理）
2. テスト（カバレッジ80%以上）
3. 可読性（JSDoc・命名規約）
4. パフォーマンス

## 禁止事項
- console.logのコミット
- 環境変数のハードコード
- テストなしのコミット
- node_modulesの直接編集
- .envファイルのgit追加

## コミットルール
- メッセージは日本語
- prefix: feat / fix / refactor / test / docs

## 詳細ルール
- API設計ガイドライン: [docs/api-guidelines.md](docs/api-guidelines.md)
- DBスキーマ定義: [docs/schema.md](docs/schema.md)

## 現在のフェーズ: MVP開発
- テストは主要パスのみ（カバレッジ目標60%）
- ドキュメントは最小限
- パフォーマンス最適化は後回し
